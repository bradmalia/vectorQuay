Phase 0 Unit Test Plan

Lightweight artifact-completeness and rule-consistency validation for Phase 0.

Versionv1.0
StatusIn Review
OwnerBrad Malia
ThemeIndigo Signal

Purpose

Phase 0 has little or no executable software, so unit testing in this phase means validating the completeness and internal consistency of individual requirements artifacts.

Test Goal

Ensure each Phase 0 artifact contains the required rules, outputs, and review fields before later phases depend on them.

Validation Areas

  • Required sections are present in each Phase 0 artifact.
  • Critical policies are explicitly stated, not implied.
  • Key decision outputs are listed.
  • Status, version, owner, and branding metadata are present.

Unit-Level Checks

UTP-01: PRD Completeness

Verify that the PRD contains scope, constraints, permissions matrix, protected-asset rules, quote-currency policy, mixed-control rules, and acceptance criteria.

UTP-02: TRD Completeness

Verify that the TRD contains research method, evidence standard, technical permissions matrix, enforcement architecture, security/secrets requirements, future signal-source boundary rules, exit criteria, and technical decision outputs.

UTP-03: Metadata Completeness

Verify that each document includes version, status, owner, and branding.

UTP-04: Policy Explicitness

Verify that prohibited actions and approval-gated actions are written explicitly rather than implied through omission.

UTP-05: Research Output Completeness

Verify that the TRD requires findings to record question, conclusion, source, confidence, impact on v1, and unresolved assumptions.

UTP-06: Security Baseline Completeness

Verify that zero-secrets-in-repo, config-template-only policy, and environment-separation requirements are explicitly present.

UTP-07: Dynamic Universe and Signal Boundary Completeness

Verify that the Phase 0 artifacts explicitly preserve dynamic asset discovery, the distinction between analysis universe and live trading universe, and the treatment of blockchain activity as a future signal source rather than a Coinbase spot execution tactic.

Execution Method

  • Perform manual checklist-based review of each individual artifact.
  • Record pass/fail against required sections and rule presence.
  • Store results in a durable review artifact under docs/test-results/.
  • Fix omissions before considering broader integration or regression validation complete.

Exit Criteria

  • Each Phase 0 artifact contains all required sections.
  • No critical rule is missing from its source document.
  • Each artifact is complete enough to participate in integration review.
