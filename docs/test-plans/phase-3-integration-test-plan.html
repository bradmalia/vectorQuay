Integration Test Plan

Phase 3 Integration Test Plan

Cross-layer validation for durable state, recovery, and replayable history.

Required Integration Gates

  • ITP3-01: startup recovers last-known state safely before the next refresh completes.
  • ITP3-02: a successful Coinbase refresh updates both live shell state and durable snapshot state.
  • ITP3-03: policy edits, alert-rule edits, and source changes create durable audit records.
  • ITP3-04: corrupted or missing persisted state falls back safely without crashing the shell.
  • ITP3-05: Activity consumes durable history across restarts.
  • ITP3-06: the read-only/no-polling boundary remains intact after persistence is introduced.

Pass Condition

The product can restart safely, reconstruct recent operational state, and expose audit history without weakening the Phase 2 exchange boundary.
