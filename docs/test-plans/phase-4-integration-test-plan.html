Integration Test Plan

Phase 4 Integration Test Plan

Cross-layer validation for deterministic decisions, GUI visibility, and non-bypassable risk approval.

Execution Environment and Fixtures

  • Primary execution environment: Ubuntu-first local workstation with valid Phase 3 durable-state behavior and valid Coinbase read-only connectivity.
  • Primary command baseline: dotnet restore, dotnet build, dotnet test, and dotnet run --project src/VectorQuay.App.
  • Required fixtures: valid live Coinbase snapshot, persisted durable state, approved asset universe, protected-asset example, and multiple risk-profile examples.
  • Expected evidence: command summaries, strategy/risk walkthrough captures, and result artifacts under docs/test-results/.

Required Integration Gates

  • ITP4-01: deterministic candidate decisions appear from current live read-only data.
  • ITP4-02: protected assets and hard limits reject candidates visibly and durably.
  • ITP4-03: aggressiveness profile changes alter decision posture but do not bypass hard risk.
  • ITP4-04: Activity distinguishes deterministic decision records from true account-event history.
  • ITP4-05: decision outputs are durable across restart through the Phase 3 persistence layer.
  • ITP4-06: no execution or paper-trading path is introduced.

Pass Condition

The product can generate deterministic candidate decisions, show them clearly in the shell, and enforce hard-risk outcomes without weakening the read-only boundary or blurring decisions with actual executed account history.
