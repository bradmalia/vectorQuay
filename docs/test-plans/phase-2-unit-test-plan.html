Unit Test Plan

Phase 2 Unit Test Plan

Component-level validation for Coinbase read-only client behavior, mapping, and restriction interpretation.

Versionv0.1
StatusDraft baseline
OwnerBrad Malia
ThemeIndigo Signal

Purpose

Phase 2 unit testing should prove that the smallest Coinbase-facing and mapping contracts behave deterministically before full shell integration is considered trustworthy.

Primary Coverage Targets

  • Credential-source resolution for Coinbase read-only use.
  • Coinbase auth-header and request-auth construction.
  • Balance-response mapping into app-facing portfolio models.
  • Product-response mapping into app-facing asset models.
  • Connection-health state classification.
  • Capital-rule interpretation from Coinbase balances.
  • Failure classification for missing credentials, auth failures, and partial payloads.

Required Unit Cases

  • UTP2-01: Coinbase credential resolution and precedence.
  • UTP2-02: Coinbase auth-header and request-auth formatting correctness.
  • UTP2-03: Account-balance payload mapping to current-value models.
  • UTP2-04: Product payload mapping to approved/watch/restricted asset rows.
  • UTP2-05: Connection-health classification for success, auth failure, network failure, and stale data.
  • UTP2-06: USD valuation lock with USDC secondary recognition preserved.
  • UTP2-07: Existing-funds-only capital interpretation.
  • UTP2-08: Partial or malformed Coinbase payload handling without unsafe fallback assumptions.

Execution Contract

UTP2-01: Coinbase Credential Resolution and Precedence

Verify that Coinbase credentials resolve deterministically from environment variables and the approved secret file, and that incomplete credential sets are classified safely.

Automation: Required.

Fixture: temporary secret-file variants and environment-variable overrides representing missing, incomplete, and complete credential states.

Command: dotnet test tests/<phase-2-test-project> --filter UTP2_01

Pass condition: credential state is classified correctly and incomplete credentials do not appear equivalent to valid credentials.

UTP2-02: Request Auth Construction

Verify that Coinbase auth-header and request-auth construction matches the required format for authenticated read-only requests.

Automation: Required.

Fixture: deterministic request example with known secret material and expected auth output.

Pass condition: generated auth headers or signatures match the expected result exactly.

UTP2-03: Balance Mapping

Verify that Coinbase balance payloads map into app-facing holdings, available-balance, and valuation models without dropping required fields silently.

Automation: Required.

Fixture: representative Coinbase balance payloads including USD and asset holdings.

Pass condition: mapped models preserve quantities, symbols, and available-vs-held distinctions correctly.

UTP2-04: Product Overlay Mapping

Verify that Coinbase product metadata maps cleanly into asset rows while preserving approved, watchlist, and restricted local overlays.

Automation: Required.

Fixture: representative product set containing approved assets, DOGE, and non-approved products.

Pass condition: the overlay state remains locally authoritative and Coinbase refresh does not erase it.

UTP2-05: Connection-Health Classification

Verify classification for successful refresh, auth failure, network failure, stale data, and never-loaded state.

Automation: Required.

Pass condition: each condition maps to a distinct and stable app-facing state.

UTP2-06: USD Valuation Lock

Verify that USD remains the locked valuation currency while USDC remains recognized as an approved secondary asset where required.

Automation: Required.

Pass condition: Phase 2 read models never promote USDC to the default valuation currency.

UTP2-07: Existing-Funds-Only Capital Interpretation

Verify that available capital derives only from returned Coinbase balances and never implies unsupported external funding paths.

Automation: Required.

Pass condition: no unit-tested interpretation path creates capital from off-platform assumptions.

UTP2-08: Malformed Payload Handling

Verify that partial or malformed Coinbase payloads surface understandable failures rather than unsafe fallback assumptions.

Automation: Required.

Pass condition: malformed payloads fail safely and do not produce silently trusted account state.

Pass Condition

All critical Phase 2 Coinbase read-only and mapping contracts have deterministic automated coverage and no unit-tested path implies write-capable exchange behavior.
