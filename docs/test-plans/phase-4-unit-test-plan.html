Unit Test Plan

Phase 4 Unit Test Plan

Deterministic strategy, scoring, and hard-risk rule coverage.

Execution Environment

  • Primary environment: local .NET 8 solution on the Ubuntu-first development workstation.
  • Primary command baseline: dotnet restore, dotnet build, and dotnet test.
  • Required fixtures: normalized account snapshot fixture, approved-universe fixture, protected-asset fixture, stale-data fixture, and aggressiveness-profile fixture.
  • Expected evidence: automated test output and targeted Phase 4 result notes under docs/test-results/.

Required Unit Coverage

  • UTP4-01: deterministic candidate generation is repeatable from identical inputs.
  • UTP4-02: scoring output is deterministic and explainable.
  • UTP4-03: protected-asset and approved-universe rules block disallowed candidates.
  • UTP4-04: hard-risk limits reject candidates even when their score is high.
  • UTP4-05: profile changes alter soft thresholds without bypassing hard limits.
  • UTP4-06: decision outputs serialize/persist cleanly into the durable history layer.
  • UTP4-07: inactive-trading / kill-switch posture still prevents executable approval semantics.

Pass Condition

The Phase 4 decision engine is deterministic, its hard-risk model is non-bypassable, and durable decision records are stable enough to feed later backtesting and paper-trading phases.
