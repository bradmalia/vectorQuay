Phase 3 Regression Test Plan

Baseline-protection plan for persistence without execution or secret leakage.

Protected Baselines

  • Coinbase Advanced remains the only exchange surface.
  • Read-only mode remains enforced throughout Phase 3.
  • No execution, no cancellation, and no paper-trading behavior is introduced.
  • Secrets remain external and must not enter durable state or audit payloads.
  • Startup/manual refresh remains explicit; persistence must not drift into silent polling.

Required Regression Checks

  • RTP3-01: persistence introduces no write-capable Coinbase path.
  • RTP3-02: persisted payloads exclude secrets and secret-derived plaintext.
  • RTP3-03: startup recovery does not label restored state as fresh live state incorrectly.
  • RTP3-04: local policy overlays remain authoritative after recovery and subsequent refresh.
  • RTP3-05: failure/corruption fallback retains safe shell behavior.
  • RTP3-06: Phase 2 no-polling and inactive-trading boundaries remain intact.

Pass Condition

Phase 3 adds durable local state and auditability without weakening the approved read-only, no-execution, and no-secret-leakage boundaries established in earlier phases.
